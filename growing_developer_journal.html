오늘 배운 내용: Git의 3대 영역

📌 Git은 단순히 타임라인 백업이 아니었다.

오늘 Git의 물리적, 논리적 공간의 차이를 확실하게 손으로 그리며 공부했다. 파일을 수정하는 것과 버전으로 박제하는 것 사이에 완충 지대가 있다는 것을 알았다.

각 단계의 작동 원리 일기:
  • Working Directory: 내 에디터 화면상에 존재하는 실제 편집 중인 공간.
  • Staging Area: 다음 Commit에 올라타기 전 파일들을 준비 공간에 줄 세워두는 영역.
  • Repository: 하나의 영구 스냅샷 버전으로 로컬 기록에 각인되는 저장 영역.

* 느낀점: 이 구조를 이해하고 나니 무분별한 git add .를 삼가고 목적에 충실한 깔끔한 커밋을 남길 수 있을 것 같다.

⚡
해결한 문제: JS 비동기 렌더링

📌 텅 빈 화면에 마운트되던 리스트 버그 탈출

⚠️ 아픈 에러 극복기

물류 데이터 로그들을 비동기 fetch()를 통해 불러와 렌더링하려 하였으나, 서버에서 데이터가 채 날아오기 전에 DOM을 비우고 가공하는 코드가 성급하게 실행되어 'Uncaught TypeError'를 마주했음.

💡 나의 시도와 해결

비동기 작업의 순서를 제어하기 위해 async/await로 페칭 로직을 감싸고, 페칭이 온전히 완수되어 객체가 담긴 직후에만 연쇄적으로 렌더 함수가 호출되도록 실행 루틴을 개편했음.

* 배운점: 자바스크립트의 마이크로태스크 큐와 이벤트 루프의 선후 조건이 브라우저에서 어떻게 동작하는지 깊이 이해한 소중한 날이다.
